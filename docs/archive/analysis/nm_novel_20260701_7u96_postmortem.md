## 修复建议

### P0：pacing=None 导致流水线崩溃

**方案 A（推荐 · fallback 兜底）**：在 `normalize_chapter_analysis_response` 中对 pacing=None 做 fallback 而非报错：

```python
# analyze_validators.py
for field in ("summary", "key_event", "hook_type"):
    result[field] = require_string(result.get(field), f"chapter_analysis.{field}")
# pacing 允许 fallback
pacing_val = result.get("pacing")
result["pacing"] = require_string(pacing_val, "chapter_analysis.pacing") if pacing_val is not None else "中"
```

**方案 B（schema 放宽）**：将 `pacing` 从必填改为可选字段，允许 None。

**方案 C（增强重试）**：对 pacing=None 的章节在重试时切换温度或追加提示词，强制要求输出 pacing。

### P1：stage_map 未合并

评估阶段完成后增加合并逻辑：将 5 个批次的 stage_map 合并为一条连贯的全局阶段序列，去除重叠范围和重复 turning_points。

### P1：chapter_functions 碎片化

引入受控词表或在分析后增加归并步骤。可参考 `tags` 系统的做法：定义核心标签集合 + 变体映射表，分析后自动归并。

### P1：evaluation 语义重复

5 个批次的评估结果需要去重/合并后再写入 `evaluation.yaml`，包括：
- 分析焦点：按语义聚类，合并近义条目。
- 世界观维度：同上。
- 角色候选：合并同角色不同名字的条目。
- turning_points：按章节去重，保留描述最完整的版本。

### P2：genre 回填

评估完成后将 `novel_type` 回写到 `meta.yaml` 的 `genre` 字段。
