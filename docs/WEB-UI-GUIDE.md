# Web 界面使用手册

> 本文面向使用者，介绍小说素材库 Web 界面的每个功能和操作方式。

## 启动

需要同时启动后端和前端：

```bash
# 终端 1：启动后端
cd backend && python main.py
# 服务地址: http://127.0.0.1:8000

# 终端 2：启动前端
cd frontend && npm run dev
# 打开浏览器访问: http://127.0.0.1:5173
```

> 后端依赖 `data/material.db` 和 `data/novels/` 目录中的数据。如果是第一次使用，先通过 CLI 入库小说或使用网页上传功能。

## 侧边栏导航

左侧固定导航栏，包含以下页面：

| 页面 | 用途 |
|------|------|
| 总览 | 全库统计仪表盘 |
| 素材库 | 浏览所有已入库的小说 |
| 上传小说 | 通过网页上传新小说文件 |
| 事件搜索 | 按标签或关键词检索事件 |
| 人物搜索 | 按条件筛选人物角色 |
| 标签字典 | 查看和管理全局标签体系 |
| 设置 | 配置 LLM API |

## 一、总览

进入首页看到的全局统计面板，帮助你了解素材库的整体规模。

**展示内容：**

- 四个核心数字：小说总数 / 事件总数 / 人物总数 / 标签记录数
- **事件类型 Top 10 饼图** — 一眼看出素材库中哪类事件最多（对决？日常？谈判？）
- **张力分布柱状图** — 了解高张力（T4-T5）和低张力（T1-T2）事件的比例
- **情绪 Top 10 条形图** — 情绪维度的分布情况
- **每部小说事件数对比** — 看各部小说的数据量差异

> 总览是只读页面，所有数据自动从数据库聚合，无需手动操作。

---

## 二、素材库

展示所有已入库小说的卡片列表。

**每张卡片显示：**
- 书名、作者
- 状态标签（`raw` / `formatted` / `outlined` / `complete` 等）
- 事件数
- 入库日期

**操作：** 点击任意卡片 → 进入该小说的详情页。

---

## 三、素材详情

这是信息最丰富的页面。顶部显示书名、作者、状态和基本数字，下方分为 **7 个 Tab**：

### 概览 Tab

- **Pipeline 操作面板**：4 个按钮可手动触发数据处理流程
  - **入库检查** — 检测原文文件是否就绪
  - **格式清洗** — 清理广告、繁简转换、章节标准化
  - **构建索引** — 将事件数据写入 SQLite 加速检索
  - **分析(LLM)** — 调用 LLM 生成大纲、人物、标签（需先在设置页配置 API）
  - 按钮绿色打勾 = 已完成；点击可重新执行
  - 运行中会显示加载动画，自动每 3 秒刷新状态
  - 出错时红色提示，可点击「清除错误并重试」

- **核心指标**：总章节、总事件、平均张力、高张力事件数 + 张力分布迷你图
- **可用数据**：绿点表示该维度数据已生成（大纲/世界观/人物/标签/事件/统计）

### 大纲 Tab

- **故事前提** — 一句话概括全书内核，附主题和基调标签
- **故事结构** — 时间线形式展示每一幕：幕名、章节范围、叙事弧线、关键事件、转折点
- **时间线** — 如有多条叙事线，分别展示
- **钩子网络** — 埋设章节 → 回收章节的钩子对照（含伏笔、悬念、因果铆合）
- **节奏曲线** — 按章节显示张力变化趋势（鼠标悬停查看详情）

### 世界观 Tab

按分类展示世界观设定卡片。常见分类：
- 力量体系（等级、规则、限制）
- 地理空间（地图、重要地点）
- 势力组织（阵营、关系）
- 背景传说、世界规则 等

每个分类自动识别数据结构：文本直接展示、列表逐条列出、对象展示名称+描述+附加属性。

### 人物 Tab

每个人物一张卡片：
- **角色图标**：主角(心) / 反派(剑) / 配角(人形)
- 姓名、角色类型标签、原型标签、道德光谱
- 别名列表
- 人物描述
- 特质标签（琥珀色 badge）
- **人物弧线**：横向时间线展示各阶段（阶段名 + 章节号）
- 出场次数、叙事功能

### 标签 Tab

分组展示小说级标签：
- **类型**（如：都市、玄幻、科幻）
- **主题**（如：成长逆袭、黑暗森林）
- **基调**（如：热血、冷峻、温情）
- **文笔**（如：白描、细腻）
- **写作长板**（如：对话、节奏控制）
- **套路**（如：扮猪吃虎、废柴逆袭）
- **叙事结构**（结构类型 / 视角 / 时间处理）
- **亮点** — 这本书最值得参考的地方
- **适合参考** — 写作中可以参考的具体事件

### 事件 Tab

分页列表展示所有事件。

- 每行显示：事件 ID、标题、张力值、章节名
- **点击展开**：查看完整摘要、出场人物（紫色 badge）、全部标签（按维度配色）
- 底部翻页控制，每页 30 条

### 统计 Tab

可视化统计报告：
- 数字概览（总章节/总事件/事件/章/平均张力）
- **张力分布柱状图**（ECharts）
- **事件类型饼图**（Top 12）
- **情绪分布横向条形图**
- **人物出场 Top 10** 排行
- **关键转折点时间线**
- **钩子统计**（埋设/回收数量）
- **写作技法标签**
- **节奏分析**（按幕展示事件数和平均张力）
- **结构评估**（开篇/中段/高潮/收尾的质量评价）
- **事件类型明细**（完整类型+占比条形图）

## 四、事件搜索

顶部两个模式 Tab 切换：

### 标签搜索

左侧筛选栏 + 右侧结果区，适合精确检索。

**筛选项：**
- **人物名** — 输入框，搜索包含该人物的事件
- **张力范围** — 设最小/最大值（1-5）
- **6 大标签层级**（点击层级名展开/收起）：
  - A 内容层：事件类型 / 冲突 / 赌注
  - B 人物层：关系 / 互动 / 弧光时刻 / 权力位差 / 道德光谱
  - C 情感层：情绪 / 读者感受
  - D 结构层：剧情阶段 / 情节功能 / 节奏
  - E 技法层：叙事技法 / 对话类型 / 视角 / 信息投放
  - F 物理层：空间类型 / 人数规模 / 时间天气

**操作方式：**
1. 点击标签值选中（变色+勾号），支持多选
2. 层级标题旁的橙点表示该层有选中标签
3. 点击「搜索」按钮执行
4. 「清空」一键清除所有筛选

**结果卡片展示：**
- 事件 ID、标题、张力值
- 来源小说 · 章节
- 事件摘要
- 匹配的标签（绿色高亮）
- 全部标签（按维度配色）
- 出场人物

> 多个条件默认取交集。如果交集为空，自动放宽为并集并按匹配度排序，结果区会显示「已放宽条件」提示。

### 全文搜索

- 输入关键词，回车或点击搜索
- 搜索范围：事件标题 + 事件摘要
- 结果卡片格式同上（但没有标签匹配高亮）

## 五、人物搜索

按条件筛选所有小说中的人物角色。

**筛选项（顶部横排）：**
- **人物名** — 模糊匹配
- **角色类型** — 主角 / 反派 / 配角 / 龙套
- **原型** — 从标签字典中选择（如：导师、反叛者、守护者）
- **道德光谱** — 从标签字典中选择（如：正义、灰色、黑暗）

**人物卡片展示：**
- 角色图标 + 姓名 + 角色标签 + 原型标签
- 来源小说
- 道德光谱、叙事功能、出场次数
- 心理特征（致命缺陷 / 执念 / 软肋 / 误信）
- **人物弧线**（可展开）：阶段 → 状态 → 触发事件 → 章节号，时间线形式

## 六、标签字典

查看和管理全局标签体系，标签由 `data/tags.yaml` 维护。

**页面结构：**
- 按 6 个事件标签层级 + 小说级标签分组
- 每个维度可点击展开，查看描述和所有标签值
- 标签值后面的数字是该标签在数据库中的使用次数

**管理操作：**

| 操作 | 怎么做 |
|------|--------|
| 新增标签 | 展开某个维度 → 点击「+」→ 输入新值 → 确认 |
| 合并标签 | 展开某个维度 → 点击「合并」→ 选择源标签和目标标签 → 确认 |

> 合并操作会同步更新 SQLite 数据库中所有引用该标签的记录。

## 七、上传小说

通过浏览器上传新小说文件。

**操作步骤：**
1. 将文件拖入虚线框，或点击选择文件
2. 支持格式：`.txt` / `.md` / `.epub`
3. 填写书名（默认从文件名提取）和作者（默认「未知」）
4. 点击「上传」

**上传成功后：**
- 显示分配的素材 ID（如 `nm_novel_20260409_a1b2`）
- 可直接点击跳转到详情页
- 在详情页的概览 Tab 中触发 Pipeline 进行后续处理

---

## 八、设置

配置 LLM API，Pipeline 中的「分析(LLM)」阶段需要此配置。

**需要填写：**
- **Base URL** — API 地址（如 `https://api.openai.com/v1`）
- **API Key** — 密钥（输入框支持显示/隐藏）
- **模型名** — 使用的模型（如 `gpt-4o`）

**操作：**
- 「保存」— 同时存储到浏览器 localStorage 和后端配置文件
- 「测试连接」— 发送请求验证 API 可用性，成功显示绿色勾号

## 常见操作流程

### 入库一本新小说

1. 进入「设置」→ 配置 LLM API → 保存
2. 进入「上传小说」→ 拖入文件 → 填信息 → 上传
3. 跳转到详情页 → 概览 Tab
4. 依次点击 Pipeline 按钮：入库检查 → 格式清洗 → 构建索引 → 分析(LLM)
5. 等待分析完成（LLM 分析可能需要几分钟）
6. 切换到各 Tab 查看生成的大纲、人物、标签等

> Pipeline 中的入库检查、格式清洗、构建索引是本地操作，秒级完成。分析(LLM) 会调用外部 API，耗时取决于小说长度和 API 速度。

### 查找参考事件

1. 进入「事件搜索」
2. 选择「标签搜索」模式
3. 展开相关层级，点选标签（如：情绪=紧张，事件类型=对决）
4. 可选：输入人物名 / 设张力范围
5. 点击搜索
6. 浏览结果卡片，查看摘要和标签匹配情况

### 了解某部小说

1. 进入「素材库」→ 点击对应小说卡片
2. 大纲 Tab → 看故事前提和结构
3. 人物 Tab → 看角色体系和弧线
4. 统计 Tab → 看全书数据分析
