# 事件拆分方案反思

## 日期
2026-04-27

---

## 一、问题发现

### 用户反馈
"所有的问题根源只有一个：事件的边界定义不清晰。"

### 问题表现

在运行 pipeline 过程中，事件拆分产出存在以下问题：

| 问题层面 | 具体表现 |
|----------|----------|
| 字段边界 | Schema 定义与实际产出不一致 |
| 粒度边界 | 同为"主线事件"，粒度差距可达33倍 |
| 覆盖边界 | 同一段文本被多个事件覆盖，但没有层级关系定义 |
| 校验结果 | h1x2 的30个事件文件全部校验失败 |

---

## 二、问题根源分析

### 核心矛盾

**"戏剧动作"是一个文学概念，不是可程序检验的客观标准。**

LLM 对这个概念的理解必然有偏差，且偏差无法通过校验脚本发现。

| 缺失内容 | 导致的问题 |
|----------|-----------|
| 粒度锚点 | 没有定义"事件的最小/最大单元" |
| 层级关系 | 没有定义"主线事件与子事件的关系" |
| 重叠处理 | 没有定义"同文本被多线索覆盖时如何标记" |
| 结构角色 | 没有定义"转折/高潮等标签的判断标准" |

---

## 三、方案评估

### 方案A：坚持事件拆分

| 评估 | 结论 |
|------|------|
| 可控性 | 不可控：边界定义模糊，LLM理解偏差无法检验 |
| 投入产出 | 投入大，产出有限 |
| 结论 | 不推荐 |

### 方案B：改用场景拆分

| 评估 | 结论 |
|------|------|
| 边界定义 | 比"事件"更客观（地点切换、时间跳跃、焦点切换） |
| 数量膨胀 | 实测：网络小说每章平均1-3个场景，不膨胀 |
| 用户反馈 | "很难从中提取规律" |
| 结论 | 场景切分本身不产生规律，只是把文本切得更细 |

### 方案C：只做章节切分+摘要

| 评估 | 结论 |
|------|------|
| 可控性 | 完全可控：章节切分是机械操作 |
| 摘要质量 | 半可控：LLM偏差，但摘要不要求精确 |
| Agent可用性 | 能看到开局范式、推进范式、转折范式、节奏范式 |
| 结论 | 最简可行方案 |

---

## 四、核心结论

### 拆分悖论

**拆分越精细，边界越难定义；拆分越粗，规律提取越依赖理解。**

| 拆分层级 | 边界困难 | 规律提取困难 |
|----------|----------|--------------|
| 事件 | 戏剧动作模糊（不可控） | 需要理解事件角色 |
| 场景 | 焦点切换信号（半可控） | 需要理解场景关系 |
| 章节 | 无困难（完全可控） | 需要理解章节内容 |

### 规律提取的本质

规律不是从拆分中自动产生的，而是从**理解内容**中产生的。

例如：
- "主线中插入配角视角，制造悬念"——这是叙事技法，需要理解场景之间的关系
- "开局困境→主动自救"——这是开局范式，需要理解人物状态变化
- "家庭团聚vs家破人亡"——这是对比手法，需要理解情节含义

这些理解，Agent 可以自己完成，不需要人为标注。

## 五、建议方案

### 最简方案

**只做章节切分 + 内容摘要。**

| 操作 | 可控性 | 产物 |
|------|--------|------|
| 章节切分 | 完全可控（机械） | 章节文本 |
| 内容摘要 | 半可控（LLM偏差） | 每章50-100字摘要 |
| 人物名提取 | 可控（名词检测） | 出场人物列表 |

### 不做的内容

- 不做事件拆分（边界不可控）
- 不做场景拆分（用户反馈难提取规律）
- 不做结构角色标注（转折/高潮定义模糊）
- 不做张力曲线标注（无法检验）

### Agent如何使用产物

Agent从章节序列+摘要中可以看到：

| 能看到 | 用途 |
|--------|------|
| 第一章摘要：入狱→困境→自救思路 | 开局范式 |
| 第1-5章摘要：破案过程 | 推进范式 |
| 第7章摘要：家人团聚+配角自杀悬念 | 转折范式 |
| 情节跨越的章节数 | 节奏范式 |

**Agent 自己有能力从摘要序列中理解结构，不需要人为标注。**

## 六、后续行动

### 如果确认最简方案

- 调整 Skills，移除事件拆分相关流程

### 第一章结构

- 整章在监牢内，连续叙事
- 人物进出（狱卒、许新年）不构成场景切换
- 只有时间跳跃和地点切换是有效的切分信号

### 第七章结构（有焦点切换）

| 段落 | 内容 | 焦点 |
|------|------|------|
| 行1-40 | 许七安与家人见面 | 许七安 |
| 行41-57 | 许新年独自准备自杀 | **焦点切换** |
| 行58-80 | 许七安回到许府发现自杀 | **焦点切换** |

焦点切换能捕捉叙事视角变化，但信号检测需要更精细规则，且用户反馈"难提取规律"。
