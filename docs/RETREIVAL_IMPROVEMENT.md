# 检索系统优化计划：原文定位与场景片段提取

## 问题背景

当前检索系统返回的是章节摘要（summary），而非原文内容。这在实际使用中存在局限性。

### 具体场景举例

用户想写一个"雨夜"场景，检索到相关章节摘要：

```
--- 某小说 第42章 ---
标题: 雨夜追踪
摘要: 主角在暴雨中追踪嫌疑人，发现关键线索...
```

**问题**：摘要只提供"发生了什么"，无法提供：
- 具体场景描写细节（雨夜的感官描写）
- 文笔修辞技巧（比喻、拟人等）
- 叙事节奏和镜头感
- 人物动作与环境的交互

用户无法直接借鉴原文的写法，只能知道"这章有雨夜场景"，然后手动去翻原文。

---

## 当前系统设计

### 数据存储结构

```
data/novels/{material_id}/
├── source.txt          # 原文完整内容
├── chapter_index.yaml  # 章节行号索引
├── chapters.yaml       # 章级分析结果（摘要）
└── chapters/
    └── 0042.yaml       # 单章分析数据
```

### 检索返回内容

| 字段 | 说明 |
|------|------|
| `summary` | 章节摘要（50-100字） |
| `title` | 章节标题 |
| `chapter_functions` | 功能标签 |
| `tension_level` | 张力等级 |
| `characters_appear` | 出场人物 |
| `key_plot_point` | 关键情节点 |

**原文 (`source.txt`) 不参与检索流程**。

---

## 优化方案

### 方案 A：增加"定位原文"功能

**思路**：检索返回摘要后，支持一键跳转到原文对应章节。

**实现方式**：
1. 检索结果增加"查看原文"选项
2. 根据 `material_id` + `chapter` 定位
3. 从 `chapter_index.yaml` 获取行号范围
4. 从 `source.txt` 提取对应章节完整内容

**优点**：
- 实现简单，无需改动分析流程
- 用户可看到完整章节上下文
- 不增加存储和 API 成本

**缺点**：
- 需要用户二次操作（先检索，再定位）
- 原文可能很长（几千字），不便于快速浏览
- 无法精确定位到"雨夜描写片段"在章节中的位置

---

### 方案 B：分析阶段增加"场景片段提取"

**思路**：让 LLM 在分析章节时，额外提取精彩片段并标注类型。

**新增字段示例**：

```yaml
# chapters/0042.yaml
summary: 主角在暴雨中追踪嫌疑人...
highlight_scenes:
  - type: "环境描写"
    excerpt: "雨水像断了线的珠子砸在青石板上，溅起的水雾模糊了街灯的光晕..."
    tags: ["雨夜", "氛围", "悬疑"]
    start_line: 156
    end_line: 162
  - type: "动作场面"
    excerpt: "他猛地转身，刀锋划破雨幕..."
    tags: ["战斗", "紧张"]
```

**检索效果**：
- 检索"雨夜"直接返回相关片段
- 无需跳转原文，直接展示借鉴内容
- 片段有标签分类，便于精准筛选

**优点**：
- 检索结果更精准，直接返回可借鉴片段
- 片段短小（100-200字），便于快速浏览
- 支持按片段类型过滤（环境描写 / 动作场面 / 心理刻画）

**缺点**：
- 需要改动分析流程，增加 LLM 调用复杂度
- 增加 API 成本（每章多提取几个片段）
- 片段提取质量依赖 LLM 判断，可能有遗漏或偏差
- 存储成本增加（每章多几个片段对象）

---

### 方案 C：A + B 结合

**思路**：两种方案并存，适应不同需求。

- **方案 B** 用于快速定位精彩片段
- **方案 A** 用于需要完整上下文的场景

**优先级建议**：
1. 先实现方案 A（低成本、快速见效）
2. 后续迭代方案 B（提升检索精准度）

---

## 后续优化方向

### 短期（方案 A）

1. 新增 CLI 命令：`novel-material read {material_id} --chapter 42`
2. 检索结果增加"定位原文"提示
3. 支持按行号范围读取片段

### 中期（方案 B）

1. 扩展章级分析 prompt，增加片段提取指令
2. 设计片段存储 schema（`chapters/*.yaml` 新增字段）
3. 建立片段标签体系（环境描写 / 动作场面 / 心理刻画 / 对话技巧）
4. 支持片段级向量检索（`excerpt_embedding`）

### 长期

1. 片段库独立存储，支持跨小说检索
2. 片段评分机制（用户反馈"这个片段很有参考价值")
3. 片段推荐（根据用户写作风格偏好推荐）
