# 标签体系总览

## 层级结构

```
L0 频道层    → 男频 / 女频 / 中性
  ↓
L1 题材层    → 玄幻 / 仙侠 / 都市 / 历史 / 科幻 / 游戏 / 悬疑 / 言情 / ...
  ↓
L2 子题材层  → 东方玄幻 / 异世大陆 / 修真文明 / 都市异能 / 架空历史 / ...
  ↓
L3 元素层    → 系统流 / 重生 / 无敌 / 废柴逆袭 / 种田 / 穿越 / ...
  ↓
L4 风格层    → 热血 / 轻松 / 虐心 / 暗黑 / 搞笑 / 爽文 / ...
```

## 标签维度分类

| 维度 | 文件 | 用途 |
|------|------|------|
| 频道分类 | channel.md | 小说大方向定位 |
| 题材分类 | genre.md | 核心题材 + 子题材 |
| 元素标签 | element.md | 核心爽点、设定元素、叙事元素 |
| 角色标签 | character.md | 人物原型、身份、关系 |
| 风格标签 | style.md | 叙事风格、情感基调、节奏 |
| 结构标签 | structure.md | 叙事结构、视角、时序 |
| 世界观标签 | setting.md | 力量体系、地理空间、势力组织 |
| 章节功能标签 | chapter-function.md | 章节在叙事中的功能定位 |

## 使用规则

1. **每部小说可选多个标签**：1 个频道 + 1-N 个题材 + N 个元素 + N 个风格
2. **题材标签互斥**：每部小说主题材只能选一个（如不能同时是"玄幻"和"仙侠"），但可以有副题材
3. **元素标签不限数量**：越多越能精确定位
4. **标签来源必须从此文件选取**：禁止自由创建标签值
5. **同义标签合并**：如果两个标签含义完全一致，使用标准名称（见各文件的"同义对照"）
