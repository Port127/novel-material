# Skill Conventions

面向**跨 agent 通用**的共享约定；只放不依赖特定 UI / 平台的规则。

## 1. metadata 约定

- `SKILL.md` frontmatter 只保留：
  - `name`
  - `description`
- 触发语义优先写进 `description`，不要依赖平台专有元数据。

## 2. 正文约定

- `SKILL.md` 只保留：
  - 任务
  - 边界
  - 输入
  - 默认执行路径
  - 输出要求
  - 关键硬约束
  - 仅在需要时读取
- 细节、示例、恢复矩阵、模式差异放进 `references/`。

## 3. 引用约定

- 优先引用**具体文件**，不要写整个目录，除非目录本身就是产物目标。
- `references/` 中的文件应当从 `SKILL.md` 直接可发现，不要做多层跳转。
- 引用 schema 时尽量列出具体 schema 文件，而不是 `docs/schemas/` 整体目录。

## 4. 执行约定

- 稳定流程优先脚本化：先脚本，后人工兜底。
- 恢复时优先看**文件系统真实状态**，再参考状态字段。
- 非破坏性流程默认直接执行，不做多轮 yes/no 停顿。

## 5. 风险约定

- 下列操作应在 skill 的高可见位置明确标注风险：
  - 删除
  - 批量改写
  - 大范围重建索引 / 数据库
  - 可能覆盖现有结果
- 真正破坏性或批量改写操作要先做影响评估，再确认。

## 6. 输出约定

跨文件变更完成后，至少报告：

- 改了哪些文件
- 验证了什么
- 还有什么风险
