# 任务

彻底删除一个素材及其关联数据，确保无残留。

## 边界

用于：
- 用户明确要求删除某个 `material_id`

不用于：
- 模糊搜索后顺手删
- 未确认情况下执行任何删除动作

## 输入

- `material_id`

## 默认执行路径

### 1. 定位目标

从 `data/index.yaml` 找到：

- 素材名称
- 状态
- 实际目录

同时检查：

- 文件夹是否存在
- SQLite 是否有残留记录
- 全局聚合索引是否包含该素材

### 2. 删除预览

预览里必须说清楚：

- 将删除哪个素材
- 影响哪些资源
- 文件夹大小与文件数
- SQLite 记录规模
- 是否存在不一致残留

### 3. 用户确认

这是**不可逆操作**。  
必须等待用户明确确认后再执行。

### 4. 执行删除

确认后按顺序处理：

1. 清 SQLite 记录
2. 更新 `data/index.yaml`
3. 删除素材目录
4. 重建全局聚合索引

### 5. 验证无残留

删除后必须验证：

- `index.yaml` 无该 ID
- 文件夹已不存在
- SQLite 记录为 0

## 输出要求

至少输出：

- 删除了哪些资源
- 是否发现并清理了残留
- 验证结果

## 关键硬约束

- 必须先预览，再等待确认
- 不允许跳过残留验证
- 删除后必须说明是否重建全局索引
