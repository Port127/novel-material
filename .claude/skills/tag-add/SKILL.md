# 任务

向 `data/tags.yaml` 增加可合法使用的新标签值。

## 边界

用于：
- 现有标签不够用
- 需要加载主题域包
- 需要给某个维度追加新值

不用于：
- 合并同义词（交给 `tag-merge`）

## 输入模式

| 调用形式 | 行为 |
|----------|------|
| `/tag-add {维度} {主题域}` | 加载域包 |
| `/tag-add {维度} custom {值}` | 加到 custom 域 |
| `/tag-add {维度} {值}` | 给扁平维度追加值 |

## 默认执行路径

### 1. 读取维度结构

先判断该维度是：

- 分层维度
- 扁平维度

### 2. 选择模式

- 分层维度优先走域包或 custom
- 扁平维度直接追加

### 3. 去重检查

新增前必须检查是否已存在相同或高度近义值。

### 4. 写回

把新值写进 `data/tags.yaml` 后，立即可用于后续标签生成。

## 输出要求

至少输出：

- 操作的是哪个维度
- 新增了哪些值
- 是域包加载还是单值追加

## 关键硬约束

- 不重复追加已有值
- 分层维度优先用域包，不滥加到核心层
- 新值命名尽量短、清晰、非同义重复
