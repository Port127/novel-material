# 任务

从写作上下文中自动提取需求，返回**事件 + 人物 + 技法**三类参考。

这是**创作型检索**入口，不是单纯的事件精确匹配。

## 边界

用于：
- “我在写师徒告别，不知道怎么写得催泪但不煽情”
- “需要一个带翻转的感情推进场景”
- “主角刚跌到低谷，想找类似桥段参考”

不用于：
- 单一关键词找素材
- 明确标签条件的直接事件检索

## 输入

- 当前章节概要
- 角色状态
- 情节目标
- 想要的读者效果
- 创作难点

## 默认执行路径

### 1. 解析上下文信号

从输入中提取：

- 事件维度
- 情感维度
- 关系维度
- 结构维度
- 技法维度
- 风格维度

### 2. 三路检索

并行组织三类结果：

1. 事件参考
2. 人物参考
3. 技法参考

优先都走 `scripts/core/search.py` 或已有索引，不直接暴力扫全库。

### 3. 组装为创作可用结果

输出不只是“找到什么”，还要解释：

- 为什么匹配当前上下文
- 可以借鉴什么
- 应该避免照抄什么

### 4. 模糊输入处理

如果上下文虽然模糊，但仍能形成一个主解释，就直接返回第一版参考结果。

只有完全无法判断时，才在最终回复里补 1-2 个最关键缺口。

## 输出要求

至少输出：

- 你对当前写作上下文的理解
- 事件参考
- 人物参考
- 技法参考
- 每类结果的“参考价值”

## 关键硬约束

- 优先脚本检索与索引检索
- 不把它退化成单纯标签匹配
- 结果必须解释“为什么对当前写作有用”
