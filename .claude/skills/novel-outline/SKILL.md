# 任务

读取小说原文，生成结构化故事大纲。

## 前置检查

1. 读取 `data/index.yaml`，确认 material_id 存在
2. 读取 `data/novels/{material_id}/meta.yaml`
3. 读取 `data/novels/{material_id}/source.txt`

## Schema

输出遵循 `docs/schemas/outline.schema.yaml`。

**注意**：本阶段识别的钩子会在 refine 阶段统一处理，写入 `hooks_network` 字段。

## 上下文控制策略

长篇小说（数百章、数百万字）无法一次放入 context window。采用**分段阅读 + 增量汇总**：

|| 小说规模 | 策略 | 每段大小 |
||----------|------|----------|
|| ≤ 50 章 | 一次性读取 | 全文 |
|| 51-300 章 | 分 3-5 段 | 约 50-100 章/段 |
|| > 300 章 | 分 5-10 段 | 约 80-150 章/段 |

**段边界优先对齐卷/幕分界**（如果原文有明确的卷划分）。

## 执行步骤

### 1. 扫描章节结构

读取 `source.txt`，**仅扫描章节标题行**（不读正文），建立章节索引：
- 总章数、各章标题、有无明确的卷/幕分界
- 据此确定分段方案（段数、每段范围）

如果存在 `chapter_index.yaml`，直接复用。

### 2. 分段阅读 + 提取

对每一段：

#### 2a. 读取本段原文

按章节索引定位，只读当前段的章节文本。

#### 2b. 提取本段要素

- **结构骨架**：本段覆盖哪个叙事阶段，核心事件，转折点
- **情节线索**：本段推进了哪些情节线
- **钩子**：本段埋设或回收了哪些钩子（记录章节号 + 描述）
  - 反转铆合：隐藏性强的线索（传统"伏笔"）
  - 悬念铆合：章末留下的疑问
  - 因果铆合：跨事件的元素关联
  - 期待铆合：建立读者期待
- **节奏感知**：标注本段内关键章节的 tension（1-5）
- **时间线**：是否存在多线叙事、闪回、时间跳转

每段产出一份**段落笔记**（内存中，不写文件），格式：

```
段落 {N}（第 {start}-{end} 章）
- 叙事阶段：{描述}
- 核心事件：[...]
- 转折点：[{chapter, description}]
- 钩子埋设：[{chapter, description, crossing_type}]
- 钩子回收：[{chapter, description}]
- 关键章节 tension：[{chapter, tension, note}]
- 情节线索推进：[...]
```

#### 2c. 进入下一段

**段间只传递段落笔记的压缩摘要**（不传原文），控制上下文累积。

### 3. 汇总合成

所有段落笔记收集完毕后，综合生成全书大纲：

#### 3a. 提取结构骨架

按幕/卷划分，标注：
- 每幕覆盖的章节范围
- 主要叙事弧线
- 核心事件
- 转折点
- 节奏特征

#### 3b. 识别时间线

如果存在多线叙事、双时间线，分别标注各线覆盖的章节。

#### 3c. 整合钩子网络

合并各段的钩子记录，匹配埋设点和回收点：
- 标注铆合形式（反转/悬念/因果/期待）
- 标注未回收的钩子
- **注意**：详细验证会在 refine 阶段完成，本阶段只做初步整合

#### 3d. 生成节奏曲线

从各段的 tension 数据中选取关键章节（开篇、中点、高潮等），生成全书节奏曲线。

### 4. 写入 outline.yaml

写入 `data/novels/{material_id}/outline.yaml`。

**注意**：本阶段识别的钩子写入 `hooks_network` 字段的初步版本，refine 阶段会补充详细验证。

### 5. 更新状态

将 `meta.yaml` 中 `status` 更新为 `outlined`（如果当前是 `raw`）。

## 输出格式

```
✅ 大纲已生成

📚 素材：{name}
🏗️ 结构：{N}幕
📈 钩子：{M}条（初步识别，refine阶段详细验证）
📁 文件：data/novels/{id}/outline.yaml

后续步骤：
  /novel-worldbuilding {id}  # 提取世界观设定
```

## 注意事项

- 大纲是全书骨架，不需要场景级细节
- premise 用一句话概括核心前提
- 钩子只记录主要的，不需要穷举
- 本阶段初步识别钩子，refine 阶段统一验证并建立铆合链
- 短篇（≤50 章）可一次性读取，无需分段
- 段落笔记只保留结构性信息，不摘抄原文，控制上下文膨胀
