# 事件切分规则

## 事件单元定义

**事件单元 = 一个完整戏剧动作**，至少能回答：

- 谁在推动动作
- 目标是什么
- 中间发生了什么
- 结果是什么

## 什么时候切分

优先在这些点切分：

1. 角色状态变化
2. 目标变化
3. 关系变化
4. 明确结果出现
5. 线索切换

## 不切分的情况

- 同一事件内部的对话轮转
- 同一事件内部的短距离地点切换
- 同一线索内部的连续推进

## 线索命名

推荐格式：

- `main`
- `romance_{角色或对象}`
- `subplot_{角色/地点/目标}`
- `cast_{角色}`
- `world_{主题}`

## 行号定位

优先级：

1. 通过摘要/结果文本在 `source.txt` 精确搜索
2. 搜索失败时按章节边界估算，并写 `lines_approximate: true`

禁止：

- 用整章起止行号冒充精确定位

## 覆盖约束

- 主线连续未覆盖章节不得超过 3
- 同章多个独立场景不能硬合并
- 回忆事件不能替代对当前正文的覆盖
