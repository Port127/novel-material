# Schema 书写规范

本项目所有 `docs/schemas/` 下的 YAML schema 文件必须遵循以下统一风格。

**参考文件**：`event-unit.schema.yaml`、`meta.schema.yaml`

## 1. 文件头

前 3-5 行为简洁的元信息：文件名、用途、路径。不加 ASCII 装饰线。

```yaml
# event-unit.schema.yaml
# 事件单元 schema
# 路径: data/novels/{material_id}/events/{event_id}.yaml
```

## 2. 字段说明段

文件头下方紧跟 `# ====== 字段说明 ======` 区块，说明：
- 该 schema 描述的对象和用途
- `R = 必填, O = 可选`
- 值域标记含义：`tags.yaml → xxx` 表示值必须从 tags.yaml 对应维度选取

```yaml
# ====================================================================
# 字段说明
# ====================================================================
#
# 事件单元 = 一个完整的戏剧动作（有起点、发展、结局）
# R = 必填, O = 可选
# 值域标记 "tags.yaml → xxx" 表示值必须从 tags.yaml 的对应维度选取
```

## 3. 行内注释格式

所有字段必须带行内注释，格式为 `# R | 描述` 或 `# O | 描述`。

**注释列必须严格对齐**。以文件中值最长的那一行为基准，确定 `#` 所在列，其余所有行的 `#` 对齐到同一列。

```yaml
id: ev_main_001                           # R | 事件ID，格式 ev{线索}_{序号}
thread: main                              # R | 叙事线索（主线/支线/感情线等）
chapters: [1, 2, 3, 4, 5]                 # R | 章节号列表（可不连续）
```

示例：如果最长的值占 36 字符，则 `#` 统一在第 42 列。

## 4. 章节分隔符

逻辑区块之间使用统一的 `# ====================================================================` 分隔线：

```yaml
# ====================================================================
# 基本信息
# ====================================================================
```

不使用 `# ──` 或 `# ---` 等变体。

## 5. 值域引用

需要从标签字典选取值的字段，标注来源：

```yaml
event_type: 推理破案                       # R | tags.yaml → event_type
moral_spectrum: 正义                       # R | tags.yaml → moral_spectrum
```

## 6. 示例值

字段使用真实、有语义的示例值，不用 `xxx` 占位（material_id 除外，可用 `nm_xxx`）。

```yaml
title: 税银案破案自救                      # R | 有语义的事件概括
summary: >-                               # R | 100-200字概括事件全过程
  许七安穿越入狱，面临流放边陲的命运。
```

## 7. 复杂概念说明区块

对于复杂概念（如 hooks、thread、粒度自适应规则），用独立的 `# ======` 区块说明：
- 概念定义
- 取值范围/枚举值
- 规则或约束

放在相关字段示例之前，作为前置说明。

## 8. YAML 安全规则

在 schema 末尾附上书写安全提醒（如引号处理、数值格式、多行标量等）。
